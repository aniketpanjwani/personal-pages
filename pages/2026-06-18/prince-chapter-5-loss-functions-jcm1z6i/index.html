Loss Functions as Probabilistic Claims

A short lesson on the key move in Chapter 5: a loss function is not just a penalty. It usually says what kind of probability distribution your model claims generated the target.

Why This Matters for LLMs

To train a language model, we repeatedly ask: given the previous tokens, how much probability did the model assign to the actual next token? Chapter 5 gives the general version of that question.

The bridge: next-token prediction is multiclass classification over the vocabulary. Cross-entropy is negative log-likelihood for the correct token.

The Core Move

Earlier chapters treat a neural network as a function that maps input x to output y. Chapter 5 changes the mental model: the network maps input x to parameters of a distribution over possible outputs.

network output: f(x, phi) = theta
probability claim: Pr(y | theta)
training objective: choose phi that makes observed y values probable

That is why loss functions differ by task. A real-valued target, a class label, a count, an angle, and a token live in different spaces, so they need different distributions.

The Recipe

  1. Identify the output domain: real number, binary label, class label, count, angle, token, or something more structured.
  2. Choose a probability distribution whose support matches that domain.
  3. Make the network output the distribution parameters, often using sigmoid, softmax, or another constraint.
  4. Train by minimizing negative log-likelihood: -sum_i log Pr(y_i | f(x_i, phi)).

Three Examples to Own

Task Distribution Loss LLM Relevance
Predict a real value Normal with predicted mean Squared error, if variance is fixed Shows how familiar losses can be likelihood assumptions
Predict yes/no Bernoulli with sigmoid probability Binary cross-entropy Useful for classifiers, reward models, and eval heads
Predict one of K classes Categorical with softmax probabilities Multiclass cross-entropy Exactly the next-token objective, with K = vocabulary size

The research habit is to ask: what probability distribution did this loss smuggle into the model? That question turns formulas into modeling assumptions.

Cross-Entropy in One Sentence

For supervised learning, cross-entropy and negative log-likelihood are the same training criterion viewed from two angles: maximize probability assigned to observed data, or minimize distance from the empirical data distribution to the model distribution.

LLM token loss = -log probability assigned to the actual next token

If the correct next token is deflation and the model assigns it probability 0.20, the token loss is -log(0.20). Better models assign higher probability to the right token, which lowers the loss.
